リニューアルしたウェブサイト全体の構成をまとめました。 今後、サイトを管理・把握するための「設計図（マップ）」としてご活用ください！

🐾 サイトのファイル構成（全体図）

サイトは以下のファイル群で成り立っています。 画像用に img フォルダを作って運用する形をおすすめします。

📁 にゃんこ自動車 Webサイト
 ├── 📄 index.html    （トップページ：ごあいさつ・選ばれる理由）
 ├── 📄 about.html    （会社概要：基本情報・スタッフ紹介）
 ├── 📄 product.html  （在庫車一覧：販売中・商談中の車を並べる）
 ├── 📄 sold.html     （販売実績：売約済みの車を並べる）
 ├── 📄 detail.html   （車両詳細：各車の詳細情報を表示する裏方ページ）
 │
 ├── 📄 style.css     （デザイン：猫耳・しっぽ・肉球・色味などの設定）
 ├── 📄 app.js        （システム：車を自動で並べたり、詳細画面を作る設定）
 ├── 📄 data.js       （データ：車の全情報がまとまったデータベース）★日常更新
 │
 └── 📁 img/          （画像用フォルダ：車の写真などをまとめて入れておく場所）

🐾 各ページ・ファイルの役割

1. 見た目のページ（HTMLファイル）

  - index.html (Home) サイトの入口です。「ごあいさつ」や「にゃんこ自動車が選ばれる理由」など、お店のこだわりを伝えるページです。
  - about.html (会社概要) お店の所在地や営業時間、代表者様や「にゃんこ部長・秘書」の紹介を掲載するページです。
  - product.html (在庫車一覧) メインの売り場です。data.js
    の中から「販売中」「商談中」の車だけを自動的にピックアップ**して並べます。ページの一番下には「販売実績（売約済）」へのリンクボタンがあります。
  - sold.html (販売実績) data.js
    の中から「売約済」の車だけを自動的にピックアップ**して並べます。お客様に「過去にこんな良い車を売っていたんだな」と安心感を持ってもらうためのページです。
  - detail.html (車両詳細)
    お客様が一覧から車をクリックしたときに開くページです。1台ごとの専用ページをわざわざ何枚も作る必要はなく、この1枚のHTMLが
    data.js からデータを受け取り、自動でその車の詳細画面に変身します。
  - お問い合わせ（外部リンク） 上部メニューの「お問い合わせ」をクリックすると、別タブで設定済みのFC2メールフォームが開きます。

2. 裏側の仕組み（CSS / JSファイル）

  - style.css (デザインの要)
    ふんわりとしたピンク・オレンジの色合い、文字の大きさ、現在のページのメニューに出る「🐾（肉球）」、見出しの「しっぽの曲線」、スマホやパソコンに合わせたレイアウト調整など、サイトのおしゃれさ・可愛さをすべてここで管理しています。
  - app.js (振り分けの要)
    「このページには販売中の車だけを出そう」「このページには詳細データを出そう」という指示を出す、サイトの頭脳です。一度設定したら基本的に触る必要はありません。
  - data.js (更新の要) 🌟一番よく使うファイル
    車の情報（名前、価格、状態、写真ファイル名、スペック、熱い思いの文章）を書き込んでおく台帳です。新しい車が入った時や、売約済みになった時は、このファイルだけを書き換えればサイト全体が自動的に更新されます。

🐾 サイトを閲覧するお客様の動き（導線）

1.  トップページ (index.html) などでお店の魅力を知る。
2.  上のメニューから 在庫車一覧 (product.html) を見る。
      - 気になる車があればクリック ➡ 詳細ページ (detail.html) で写真やコラムを読む。
      - 過去の実績が見たい場合は一番下のボタンをクリック ➡ 販売実績 (sold.html) を見る。
3.  欲しい車があったり、質問があれば、上のメニューから お問い合わせフォーム へ飛んで連絡する。

このような、お客様が迷わずに車探しを楽しめる、スッキリとしつつも温かい構成になっています！
